Processus recherché

  1. Stocker l’état des sources v1

  2. Stocker l’état courant

  3. Revenir à la v1

  4. Corriger le bug / livrer

  5. Stocker l’état v1.0.1

  6. Revenir à l’état courant

Conflits

  • Merge effectué "ligne par ligne"

  • Deux lignes différentes modifiées dans le même fichier ⇒ merge automatique

  • Même ligne = conflit ⇒ merge manuel nécessaire

Merges manuels

  • Editeur de texte pour les merges simples

  • Outils spécifiques pour les merges plus complexes

    • P4Merge par exemple

    • git mergetool

Bonnes pratiques

  • Faire une branche par fonctionnalité

Github

  • Service d’hébergement basé sur Git

  • "Social"

  • Basé sur le Fork

    • Copie du dépôt

Collaborer sur Github

  1. Copier le dépôt voulu dans l’interface Github ("forker")

  2. Cloner le dépôt copié

  3. Faire les modifications souhaitées

  4. Committer/pousser

  5. Faire une Pull Request dans l’interface Github
